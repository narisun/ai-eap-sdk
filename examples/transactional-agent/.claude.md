# transactional-agent — context for AI coding agents

This project uses **EAP-Core** for cross-cutting concerns. **Do not reimplement them.**

- **Business logic** lives in `agent.py` and `tools/*.py` only.
- **Cross-cutting concerns** (PII masking, policy enforcement, OTel observability,
  prompt-injection sanitization, output validation) are owned by `eap_core`
  middleware. Add or compose middleware in `agent.py:build_client()`.
- **Tools** are added with `eap create-tool --name <name> --mcp` (do not
  hand-write MCP server code).
- **Real cloud calls** require setting `EAP_ENABLE_REAL_RUNTIMES=1`. Default
  runs on `LocalRuntimeAdapter` with deterministic canned responses.
- **Eval** is run via `eap eval --dataset tests/golden_set.json`. The dataset
  format is `{id, input, expected_contexts, expected_answer_substrings}`.

When you change `agent.py`, also update `configs/agent_card.json` if the
advertised skills change. Or call `build_card(name=..., skills_from=registry)`
at startup to keep them in sync automatically.
