# v1.3.0 Implementation Plan — Transport completeness (SSE + WebSocket + identity-aware HTTP auth)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Complete the MCP client's transport story by adding the two remaining wire formats (legacy SSE and WebSocket) and wiring the existing EAP-Core identity layer into the HTTP/SSE transports via a `BearerTokenAuth` httpx adapter. After v1.3, the client supports all four MCP transports the upstream `mcp` package ships (stdio, http, sse, websocket) and a user can plug a `NonHumanIdentity` directly into a server config without writing their own httpx.Auth subclass.

**Architecture:** Strict additive minor on top of v1.2.1. `McpServerConfig.transport` Literal extends from `["stdio", "http"]` to `["stdio", "http", "sse", "websocket"]`. Pool dispatcher gains two new branches (`_spawn_sse`, `_spawn_websocket`) mirroring the v1.2 `_spawn_http` pattern; both new transports return 2-tuples so the existing `_unpack_transport_streams(arity=2)` helper handles them. New `BearerTokenAuth(httpx.Auth)` class lives in a small `eap_core.mcp.client.auth` module; it accepts any object exposing the EAP-Core `IdentityToken` protocol (`get_token(audience, scope)`) and attaches `Authorization: Bearer <token>` to every request. The identity layer itself is unchanged.

**Tech Stack:** Python 3.11+, pydantic v2 (model_validator update for the broadened Literal), `mcp>=0.9` (the new transport functions ship in current versions), `httpx>=0.27` (already a core dep — `httpx.Auth` is the integration seam). No new optional extras; SSE and WebSocket sit under the existing `[mcp]` extra alongside Streamable-HTTP.

**Scope discipline:**
- IN: Legacy SSE transport; WebSocket transport (URL-only — upstream `websocket_client` takes no auth or headers); identity-aware HTTP/SSE auth via `BearerTokenAuth`; integration tests for the new transports (in-process FastMCP server where it supports them, mocked-httpx otherwise); model_validator updated for the broadened Literal.
- OUT (deferred to v1.4+): WebSocket auth (gated on upstream `websocket_client` adding `headers` / `auth` parameters); deeper JSON-Schema validation (Phase 3); AsyncExitStack partial-unwind for `pool.reconnect` (Phase 3); bare `tests/` collision (Phase 3); H8 Cedar live tests; H18/H19 cloud live-runtime tests (Phase 4).

**Backward compat:** Strict additive. The Literal extension widens the accepted values — existing stdio + http configs are unchanged. The `BearerTokenAuth` class is a new public export; nothing existing renames or moves.

**Upstream signatures (verified during plan drafting):**

```python
mcp.client.sse.sse_client(
    url: str,
    headers: dict[str, Any] | None = None,
    timeout: float = 5,
    sse_read_timeout: float = 300,
    httpx_client_factory: McpHttpClientFactory = create_mcp_http_client,
    auth: httpx.Auth | None = None,
    on_session_created: Callable[[str], None] | None = None,
) -> AsyncGenerator[tuple[ReadStream, WriteStream], None]  # 2-tuple — arity=2

mcp.client.websocket.websocket_client(
    url: str,
) -> AsyncGenerator[tuple[ReadStream, WriteStream], None]  # 2-tuple — arity=2
```

Key differences from `streamable_http_client`:
- SSE returns a 2-tuple (no `get_session_id`). Use `_unpack_transport_streams(arity=2)` — no new arity needed.
- SSE keeps the old-style `headers` / `auth` kwargs (matches what `streamable_http_client` used to be before its rename). We pass `cfg.headers` and `cfg.auth` through directly.
- WebSocket takes only URL. No auth, no headers. **Cannot authenticate WebSocket MCP servers via standard means until upstream adds parameters.** v1.3 documents this limitation; v1.4 picks it up when upstream catches up.

```python
@model_validator(mode="after")
def _validate_transport_fields(self) -> McpServerConfig:
    if self.transport == "stdio":
        if not self.command:
            raise ValueError("transport='stdio' requires command (the executable to run)")
        for field_name in ("url", "headers", "auth"):
            if getattr(self, field_name) is not None:
                raise ValueError(f"transport='stdio' forbids {field_name!r} (http/sse-only field)")
    elif self.transport in ("http", "sse"):
        if not self.url:
            raise ValueError(f"transport={self.transport!r} requires url (the MCP server endpoint)")
        for field_name in ("command", "cwd", "env"):
            if getattr(self, field_name) is not None:
                raise ValueError(f"transport={self.transport!r} forbids {field_name!r} (stdio-only field)")
        if self.args:
            raise ValueError(f"transport={self.transport!r} forbids 'args' (stdio-only field)")
    elif self.transport == "websocket":
        if not self.url:
            raise ValueError("transport='websocket' requires url (the MCP server endpoint)")
        for field_name in ("command", "cwd", "env", "headers", "auth"):
            if getattr(self, field_name) is not None:
                raise ValueError(
                    f"transport='websocket' forbids {field_name!r} — the upstream "
                    f"``websocket_client`` takes only url; auth via URL query params if "
                    f"needed (no clean Authorization header convention in WebSockets)"
                )
        if self.args:
            raise ValueError("transport='websocket' forbids 'args' (stdio-only field)")
    return self
```

- [ ] **Step 1.2: Add `_spawn_sse` to the pool**

```python
async def _spawn_sse(self, cfg: McpServerConfig) -> McpServerHandle:
    """Open a legacy SSE session against a remote MCP server.

    Unlike ``streamable_http_client``, ``sse_client`` keeps the original
    keyword API: ``headers``, ``timeout``, ``auth`` are passed directly.
    Returns a 2-tuple ``(read, write)`` — no session-id callback to drop.
    """
    try:
        from mcp.client.sse import sse_client
    except ImportError as e:
        raise McpServerSpawnError(
            "MCP client requires the [mcp] extra: pip install eap-core[mcp]"
        ) from e

    assert self._stack is not None, "pool not entered"
    assert cfg.url is not None  # enforced by McpServerConfig validator
    transport_cm = sse_client(
        cfg.url,
        headers=cfg.headers,
        auth=cfg.auth,
        timeout=min(cfg.request_timeout_s, 30.0),
    )
    return await self._open_session(cfg, transport_cm, arity=2)
```

Note the `timeout=min(cfg.request_timeout_s, 30.0)` choice: `sse_client`'s `timeout` parameter is the **connection** timeout (defaults to 5s upstream). Our `request_timeout_s` is the per-call timeout used by `McpClientSession.call_tool`. The two are different concepts; for the SSE connect step we use `min(request_timeout_s, 30)` to avoid hanging indefinitely while still respecting an aggressively-low per-call setting.

## Task 3: Identity-aware HTTP/SSE auth — `BearerTokenAuth`

**Why:** Today users pass `auth=httpx.BasicAuth(...)` or a custom `httpx.Auth` instance to authenticate HTTP/SSE MCP transports. EAP-Core's existing identity layer (`NonHumanIdentity`, `LocalIdPStub`, `VertexAgentIdentityToken`, etc.) already exposes a `get_token(audience, scope)` Protocol — there's no clean way to plug that into HTTP auth without users writing their own `httpx.Auth` subclass. `BearerTokenAuth` closes that gap: it accepts any object satisfying the `IdentityToken` Protocol and attaches `Authorization: Bearer <token>` to every request, with the token fetched per-request through the existing identity layer's caching/refresh logic.

**Files:**
- Create: `packages/eap-core/src/eap_core/mcp/client/auth.py`
- Modify: `packages/eap-core/src/eap_core/mcp/client/__init__.py` (export `BearerTokenAuth`)
- Modify: `packages/eap-core/src/eap_core/mcp/__init__.py` (re-export at top level)
- Create: `packages/eap-core/tests/test_mcp_client_auth.py`

### Subtasks

- [ ] **Step 3.1: `auth.py`**

```python
"""``httpx.Auth`` adapter that fetches Bearer tokens from an EAP-Core
identity layer.

EAP-Core ships a small ``IdentityToken`` Protocol (see
``eap_core.identity``) that every workload-identity implementation
satisfies — ``NonHumanIdentity``, ``LocalIdPStub``,
``VertexAgentIdentityToken``, ``OIDCTokenExchange``, etc. The Protocol
is one method: ``get_token(audience, scope) -> str``.

``BearerTokenAuth`` wraps any such object as an ``httpx.Auth`` flow
that attaches ``Authorization: Bearer <token>`` to every outgoing
request. Token refresh and caching are the identity layer's
responsibility — this class is intentionally trivial; it calls
``get_token`` once per request and assumes the implementation has
sensible caching semantics.
"""

from __future__ import annotations

from collections.abc import Generator
from typing import TYPE_CHECKING, Any

if TYPE_CHECKING:
    import httpx


class BearerTokenAuth:
    """Plug an EAP-Core identity into ``httpx`` authentication.

    Usage::

        from eap_core.identity import NonHumanIdentity
        from eap_core.mcp.client import (
            BearerTokenAuth, McpClientPool, McpServerConfig,
        )

        identity = NonHumanIdentity(...)
        cfg = McpServerConfig(
            name="bankdw",
            transport="http",
            url="https://bankdw.example.com/mcp",
            auth=BearerTokenAuth(
                identity,
                audience="mcp.bankdw.example.com",
                scope="read",
            ),
        )

    The ``IdentityToken`` Protocol is duck-typed — any object with
    ``get_token(audience, scope) -> str`` works. The protocol stays
    decoupled from any specific identity implementation so a future
    JWT-only or SPIFFE-issued identity slots in without changes here.
    """

    name = "bearer_token"

    def __init__(
        self,
        identity: Any,
        *,
        audience: str | None = None,
        scope: str = "",
    ) -> None:
        # ``identity`` is typed ``Any`` rather than ``IdentityToken``
        # so this module doesn't hard-import the identity package and
        # so tests can pass stub identities without subclassing the
        # Protocol. Runtime requirement: object with
        # ``get_token(audience, scope) -> str``.
        if not callable(getattr(identity, "get_token", None)):
            raise TypeError(
                "BearerTokenAuth: identity must have a callable .get_token(audience, scope) method"
            )
        self._identity = identity
        self._audience = audience
        self._scope = scope

    def sync_auth_flow(self, request: "httpx.Request") -> Generator["httpx.Request", "httpx.Response", None]:
        """``httpx.Auth`` sync flow: attach token then forward."""
        token = self._identity.get_token(audience=self._audience, scope=self._scope)
        request.headers["Authorization"] = f"Bearer {token}"
        yield request

    def async_auth_flow(self, request: "httpx.Request") -> Generator["httpx.Request", "httpx.Response", None]:
        """``httpx.Auth`` async flow — token fetch is sync at the
        identity layer (the existing identity implementations all
        return strings, not awaitables); we just attach and forward.
        """
        token = self._identity.get_token(audience=self._audience, scope=self._scope)
        request.headers["Authorization"] = f"Bearer {token}"
        yield request
```

Note: `httpx.Auth`'s protocol expects subclasses to implement either or both of `auth_flow` / `sync_auth_flow` / `async_auth_flow`. Defining BOTH `sync_auth_flow` and `async_auth_flow` covers httpx's sync and async clients without requiring `BearerTokenAuth` to subclass `httpx.Auth` (which would pull httpx into the module-level imports). The duck-typed signature works because httpx checks for the method names, not the class hierarchy.

If the upstream `streamable_http_client` (or `sse_client`) requires an actual `httpx.Auth` instance (via isinstance check), we'd need to inherit. **Step 3.2 verifies this** — if isinstance is required, switch the class to `class BearerTokenAuth(httpx.Auth)` and import `httpx` at module level (httpx is already a core dep, so the import is fine; the lazy-import dance was to avoid `Any` annotation noise, not to dodge the import).

```python
"""Tests for BearerTokenAuth — identity-aware httpx auth flow."""

from __future__ import annotations

import pytest
from unittest.mock import Mock

from eap_core.mcp.client import BearerTokenAuth


class _StubIdentity:
    """Minimal IdentityToken Protocol stand-in."""
    def __init__(self, token: str = "fake-token"):
        self.token = token
        self.calls: list[tuple[str | None, str]] = []
    def get_token(self, *, audience: str | None = None, scope: str = "") -> str:
        self.calls.append((audience, scope))
        return self.token


def test_construction_with_valid_identity_succeeds():
    identity = _StubIdentity()
    auth = BearerTokenAuth(identity, audience="mcp.x.com", scope="read")
    assert auth.name == "bearer_token"


def test_construction_with_invalid_identity_raises_typeerror():
    """Caller passed something that doesn't have ``.get_token``."""
    with pytest.raises(TypeError, match="get_token"):
        BearerTokenAuth(object())  # no get_token method


def test_sync_auth_flow_attaches_bearer_header():
    """Sync flow: yield the request with Authorization header set."""
    identity = _StubIdentity(token="xyz")
    auth = BearerTokenAuth(identity, audience="x", scope="y")
    request = Mock(headers={})
    flow = auth.sync_auth_flow(request)
    next(flow)
    assert request.headers["Authorization"] == "Bearer xyz"
    # Verify the audience/scope passed through
    assert identity.calls == [("x", "y")]


def test_async_auth_flow_attaches_bearer_header():
    """Async flow does the same thing (the existing identity layer
    returns sync strings, not awaitables)."""
    identity = _StubIdentity(token="async-token")
    auth = BearerTokenAuth(identity, audience=None, scope="")
    request = Mock(headers={})
    flow = auth.async_auth_flow(request)
    next(flow)
    assert request.headers["Authorization"] == "Bearer async-token"


def test_token_fetched_per_request():
    """Each request triggers a fresh ``get_token`` call. Token caching
    is the identity layer's responsibility — ``BearerTokenAuth``
    doesn't add its own cache."""
    identity = _StubIdentity()
    auth = BearerTokenAuth(identity, audience="x")
    # Two requests = two get_token calls.
    for _ in range(2):
        request = Mock(headers={})
        next(auth.sync_auth_flow(request))
    assert len(identity.calls) == 2
```
