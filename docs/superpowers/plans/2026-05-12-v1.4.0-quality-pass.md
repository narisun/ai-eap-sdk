## Task 1: Per-handle `AsyncExitStack` — clean `pool.reconnect`

**Why:** v1.1 documented and v1.2-v1.3 carried forward: `pool.reconnect()` spawns a new session/subprocess but the old one isn't torn down until pool exit because `AsyncExitStack` doesn't support partial unwind. Fix by giving each handle its own nested stack; the pool's outer stack just owns the handle stacks.

**Files:**
- Modify: `packages/eap-core/src/eap_core/mcp/client/pool.py`
- Modify: `packages/eap-core/tests/test_mcp_client_pool.py` (verify reconnect tears down the old handle's resources)

### Subtasks

- [ ] **Step 1.1: Add `_stack: AsyncExitStack | None` field to `McpServerHandle`** (or wrap creation in a helper that returns the handle + its stack). Pre-populated; closed in `_close_handle()`.

- [ ] **Step 1.2: Refactor `_open_session`** to receive the per-handle stack and enter the transport context manager onto IT, not `self._stack`. The pool's `_stack` becomes a container of handle-stacks.

```python
async def _spawn(self, cfg):
    assert self._stack is not None
    handle_stack = AsyncExitStack()
    await self._stack.enter_async_context(handle_stack)
    handle = await self._dispatch_spawn(cfg, handle_stack)
    handle._stack = handle_stack  # attach for later teardown
    return handle
```

- [ ] **Step 1.3: Rewrite `reconnect`** to call `handle._stack.aclose()` before spawning fresh:

```python
async def reconnect(self, server_name: str) -> None:
    if server_name not in self._handles:
        raise KeyError(f"unknown server: {server_name!r}")
    cfg = self._handles[server_name].config
    # Close the old handle's resources cleanly. AsyncExitStack.aclose()
    # unwinds in LIFO order: ClientSession first, then the transport.
    old_handle = self._handles[server_name]
    if old_handle._stack is not None:
        await old_handle._stack.aclose()
    # Spawn the replacement onto the pool's outer stack with a fresh
    # per-handle nested stack.
    self._handles[server_name] = await self._spawn(cfg)
```

- [ ] **Step 1.4: Update pool/handle docstrings** — drop the "leaks until pool exit" caveat.

- [ ] **Step 1.5: Test the cleanup** — extend `test_reconnect_replaces_handle_with_fresh_session` (or add a new test) that asserts the old handle's stack is closed after reconnect. The stub upstream can track close calls.

- [ ] **Step 1.6: Gauntlet** (fresh caches) + commit.

## Task 2: `jsonschema`-backed output validation

**Why:** v1.1's `_maybe_validate` does shallow required-keys only. Tools advertising rich `outputSchema` (types, formats, enum constraints) silently pass invalid responses. Add proper JSON Schema validation.

**Files:**
- Modify: `packages/eap-core/pyproject.toml` (add `jsonschema>=4.0` to the `[mcp]` extra)
- Modify: `packages/eap-core/src/eap_core/mcp/client/adapter.py` (extend `_maybe_validate`)
- Modify: `packages/eap-core/tests/test_mcp_client_adapter.py` (new tests for deep validation)

### Subtasks

- [ ] **Step 2.1: Update `_maybe_validate`** to prefer `jsonschema.validate(decoded, schema)` when available; fall back to the v1.1 shallow check when jsonschema isn't installed.

```python
def _maybe_validate(decoded, *, schema, server_name, tool):
    if schema is None:
        return decoded
    try:
        import jsonschema
        try:
            jsonschema.validate(instance=decoded, schema=schema)
        except jsonschema.ValidationError as e:
            raise McpOutputSchemaError(
                tool=tool, payload=decoded, schema=schema, reason=str(e)
            ) from e
        return decoded
    except ImportError:
        pass  # fall through to shallow check
    # Shallow fallback (v1.1 behavior): required-keys presence only.
    if isinstance(decoded, dict) and "type" in schema and schema["type"] == "object":
        required = schema.get("required", [])
        missing = [k for k in required if k not in decoded]
        if missing:
            raise McpOutputSchemaError(
                tool=tool, payload=decoded, schema=schema,
                reason=f"missing required keys: {sorted(missing)}",
            )
    return decoded
```

- [ ] **Step 2.2: Add `jsonschema>=4.0` to `[mcp]` extra** in `packages/eap-core/pyproject.toml`. The `mcp` package itself doesn't already pull it in (verify); if it does, this is a no-op.

- [ ] **Step 2.3: New tests** — cover deep validation:
  - Schema with `type: string` rejects integer payload
  - Schema with `enum: [...]` rejects unlisted value
  - Schema with nested `properties` rejects type-mismatched sub-field
  - The fallback path (when jsonschema absent — monkeypatch the import) still does the v1.1 shallow check
