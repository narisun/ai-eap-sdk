#### 2.2 — Document `RealRuntimeDisabledError`

- **`docs/developer-guide.md` §3.6** — add to the exception bullet list:

  ```
  - `RealRuntimeDisabledError` (subclass of `EapError`) is raised by every cloud-runtime adapter and integration method when `EAP_ENABLE_REAL_RUNTIMES=1` is not set. Use `except RealRuntimeDisabledError` to selectively catch stub-mode failures without intercepting genuine `NotImplementedError` (which only fires on truly-unimplemented paths).
  ```

- **`docs/user-guide-aws-agentcore.md` Troubleshooting** (around L869) — replace the `NotImplementedError: AgentCore adapter requires...` entry with:

  ```
  **`RealRuntimeDisabledError: AgentCore adapter requires the [aws] extra
  and AWS credentials. Set EAP_ENABLE_REAL_RUNTIMES=1 once configured.`**

  You forgot the env flag. The flag is intentional — it prevents tests
  from accidentally hitting AWS. Set `EAP_ENABLE_REAL_RUNTIMES=1` in
  your runtime env (not in pytest). `RealRuntimeDisabledError` is an
  `EapError` subclass, so `except EapError` catches it consistently
  with the rest of the SDK's safety errors.
  ```

- **`docs/user-guide-gcp-vertex.md` Troubleshooting** — same shape, Vertex flavor.

#### 2.3 — Document `PolicyConfigurationError`

- **`docs/developer-guide.md` §3.6** — add to the exception bullet list:

  ```
  - `PolicyConfigurationError` (subclass of `EapError`) is raised by `PolicyMiddleware` when `ctx.metadata["policy.action"]` or `["policy.resource"]` is not populated. `EnterpriseLLM.generate_text/stream_text/invoke_tool` always set these slots from trusted SDK-internal sources; custom-pipeline users wiring `PolicyMiddleware` directly must populate them before passing the request.
  ```

- **Both user-guide troubleshooting sections** — add:

  ```
  **`PolicyConfigurationError: PolicyMiddleware called without ctx.metadata['policy.action'] set ...`**

  You constructed a custom `MiddlewarePipeline` and routed a request
  through `PolicyMiddleware` without going through `EnterpriseLLM`.
  Set `ctx.metadata["policy.action"]` and `["policy.resource"]`
  explicitly from a trusted source (the tool name for invoke_tool,
  `"generate_text"` for generate_text). This is the defense-in-depth
  closure of H9 — `req.metadata` is caller-mutable and is no longer
  trusted by the policy middleware.
  ```

#### 2.4 — Document `.eapignore` `!`-negation + expanded skip-dirs

- **`docs/user-guide-aws-agentcore.md` §1.17** (around the `.eap-manifest.txt` mention, L471) — add:

  ```
  **`.eapignore` syntax:** lines are deny patterns by default (shell-style
  globs against the file's relative path). Lines starting with `#` are
  comments. Lines starting with `!` are *allow* patterns that re-include
  paths the default deny-list or skip-dirs would have excluded. Example:

  ```
  # Exclude a build artifact directory
  build/cache/

  # Re-include dist/ even though it's in _DEFAULT_SKIP_DIRS — useful if
  # your project legitimately ships a static-bundle dist/ folder.
  !dist/
  ```

  `_DEFAULT_SKIP_DIRS` covers common build-cache patterns out of the box:
  `dist`, `.venv`, `venv`, `__pycache__`, `.pytest_cache`, `.mypy_cache`,
  `.ruff_cache`, `node_modules`, `.terraform`, `.next`, `.nuxt`, `.cache`,
  `build`, `target`, `.tox`, `.coverage`, `htmlcov`, `.eap`.
  ```
