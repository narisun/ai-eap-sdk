## Task 1: `on_tool_call` Protocol hook + policy re-evaluation

**Goal:** Add an explicit lifecycle hook for tool invocation. Middlewares may transform `args` via the hook; the SDK MUST re-evaluate policy after mutation so authorization decisions can't be bypassed.

**Files:**
- Modify: `packages/eap-core/src/eap_core/middleware/base.py` — add `on_tool_call` to Protocol + PassthroughMiddleware
- Modify: `packages/eap-core/src/eap_core/middleware/pipeline.py` — new `run_tool` method (parallel to `run`/`run_stream`)
- Modify: `packages/eap-core/src/eap_core/client.py:invoke_tool` — invoke the new pipeline path
- Modify: `packages/eap-core/tests/test_pipeline.py` — add no-op `on_tool_call` to direct-Protocol mocks
- Test: `packages/eap-core/tests/test_middleware_tool_call.py` (new)

**Design:**

New Protocol method:

```python
async def on_tool_call(
    self,
    tool_name: str,
    args: dict[str, Any],
    ctx: Context,
) -> dict[str, Any]: ...
```

Returns the (possibly modified) args dict. Middlewares may mutate or return as-is.

`MiddlewarePipeline.run_tool(tool_name, args, ctx, terminal)`:

1. Run `on_request` L→R (constructed Request has the args in metadata for observability/audit, but pipeline reads the canonical args from a separate parameter)
2. Run `on_tool_call` L→R, each middleware may transform `args`
3. **CRITICAL: re-evaluate `PolicyMiddleware` against the post-mutation args** by updating `ctx.metadata["policy.action"]` / `["policy.resource"]` from the (potentially mutated) args, then call `PolicyMiddleware.on_request` AGAIN explicitly. This closes the v1.7 deferred concern: silent mutation no longer bypasses policy.
4. Call terminal with the final args
5. Run `on_response` R→L
6. Run `on_call_end` R→L (mirrors `run()`)

If a middleware's `on_tool_call` raises, fall through to standard `on_error` + `on_call_end` semantics (same as `run`/`run_stream`).

Wait — re-evaluating policy by calling `on_request` again is awkward. Cleaner design: introduce a small `_reauthorize` helper that walks `self._mws` looking for a middleware that exposes a `requires_reauthorize_for_tool_call` flag (or simpler: just re-run any middleware named `policy` in the chain). Actually, the cleanest approach is:

**Policy middleware reads `ctx.metadata["policy.action"]` and `["policy.resource"]` already.** If we update those metadata keys after `on_tool_call` mutations, then re-running ONLY the policy middleware's `on_request` is sufficient. The pipeline can identify the policy middleware by `isinstance(mw, PolicyMiddleware)` or by `mw.name == "policy"`.

Simpler still: have the pipeline call a `Middleware.on_tool_call_post_mutation(tool_name, args, ctx) -> None` hook that PolicyMiddleware overrides. Default no-op. PolicyMiddleware's override re-runs the authorization check against the now-current `policy.action` / `policy.resource`. This avoids forcing the pipeline to know about PolicyMiddleware specifically.

**Final design — minimal Protocol additions:**

```python
# base.py
async def on_tool_call(
    self, tool_name: str, args: dict[str, Any], ctx: Context
) -> dict[str, Any]: ...
async def on_tool_call_post_mutation(
    self, tool_name: str, args: dict[str, Any], ctx: Context
) -> None: ...
```

PassthroughMiddleware defaults: `on_tool_call` returns `args` unchanged; `on_tool_call_post_mutation` is no-op.

PolicyMiddleware overrides `on_tool_call_post_mutation` to re-run its policy check against `ctx.metadata` keys (which the SDK updates from the mutated args before calling this hook).

`EnterpriseLLM.invoke_tool` calls `pipeline.run_tool(...)` which orchestrates:

```python
async def run_tool(
    self,
    tool_name: str,
    args: dict[str, Any],
    ctx: Context,
    terminal: Callable[[str, dict[str, Any], Context], Awaitable[Any]],
) -> Any:
    ran: list[Middleware] = []
    try:
        try:
            # Phase 1: standard request hooks (audit, threat-detection, etc.)
            req = _build_tool_request(tool_name, args)
            for mw in self._mws:
                ran.append(mw)
                req = await mw.on_request(req, ctx)

            # Phase 2: tool-call mutation phase, L→R
            for mw in self._mws:
                args = await mw.on_tool_call(tool_name, args, ctx)

            # Phase 3: SDK updates trusted policy inputs from current args
            # (action/resource may now be derived differently — e.g., a
            # middleware that resolves a tool alias would have rewritten
            # tool_name effectively through args/metadata).
            # The base policy.action/policy.resource were set by
            # EnterpriseLLM._prepare_call_context based on the ORIGINAL
            # tool_name + args. We re-stamp now so policy_post_mutation
            # sees the current state.
            ctx.metadata["policy.action"] = f"tool:{tool_name}"
            ctx.metadata["policy.resource"] = tool_name
            # (Future enhancement: derive resource from args field if
            # documented contract emerges.)

            # Phase 4: post-mutation hooks — PolicyMiddleware re-authorizes here
            for mw in self._mws:
                await mw.on_tool_call_post_mutation(tool_name, args, ctx)

            # Phase 5: terminal call
            result = await terminal(tool_name, args, ctx)

            # Phase 6: response hooks (audit, observability)
            resp = Response(text=str(result), payload=result)
            for mw in reversed(ran):
                resp = await mw.on_response(resp, ctx)
            return resp.payload
        finally:
            for mw in reversed(ran):
                try:
                    await mw.on_call_end(ctx)
                except Exception as secondary:
                    _LOG.warning(...)
    except Exception as exc:
        await self._on_error(ran, exc, ctx)
        raise
```

This preserves the security invariant: any mutation triggers a re-authorization. The PolicyMiddleware decides — it's not the pipeline's job to know what authorization means.

## Task 2: `OutputValidationMiddleware` modes (P2-10)

**Goal:** Add `mode: Literal["strict_json", "extract_json", "provider_native"]` to `OutputValidationMiddleware`. Default stays `strict_json` for backward compat; users opt into the more forgiving modes.

**Files:**
- Modify: `packages/eap-core/src/eap_core/middleware/validate.py`
- Test: `packages/eap-core/tests/test_output_validation_modes.py` (new)

**Design:**

```python
class OutputValidationMiddleware(PassthroughMiddleware):
    """Validates `resp.text` against a Pydantic schema set via `schema=` kwarg.

    Modes:
    - ``strict_json`` (default): expects `resp.text` to be parseable as JSON.
    - ``extract_json``: extracts the first JSON object/array from `resp.text`
      via regex (handles fenced ```json blocks and JSON-with-prose).
    - ``provider_native``: trusts the adapter's `resp.payload` (set by the
      adapter when the provider supports native structured output). Falls
      back to strict_json if `resp.payload` is None.
    """

    name = "output_validation"

    def __init__(self, mode: Literal["strict_json", "extract_json", "provider_native"] = "strict_json") -> None:
        self._mode = mode

    async def on_response(self, resp: Response, ctx: Context) -> Response:
        schema: type[BaseModel] | None = ctx.metadata.get("output_schema")
        if schema is None:
            return resp
        data = self._extract_data(resp)
        try:
            payload = schema.model_validate(data)
        except ValidationError as e:
            raise OutputValidationError(errors=e.errors()) from e
        return resp.model_copy(update={"payload": payload})

    def _extract_data(self, resp: Response) -> Any:
        if self._mode == "provider_native":
            if resp.payload is not None:
                return resp.payload
            # Fall back to strict_json
        if self._mode == "extract_json":
            return self._extract_first_json(resp.text)
        # strict_json (default)
        try:
            return json.loads(resp.text)
        except json.JSONDecodeError as e:
            raise OutputValidationError(errors=[{"type": "json_decode", "msg": str(e)}]) from e

    def _extract_first_json(self, text: str) -> Any:
        """Pull the first JSON object/array from text (regex over fenced blocks first)."""
        # Try fenced ```json block first
        fenced_match = re.search(r"```(?:json)?\s*(.+?)```", text, re.DOTALL | re.IGNORECASE)
        if fenced_match:
            candidate = fenced_match.group(1).strip()
            try:
                return json.loads(candidate)
            except json.JSONDecodeError:
                pass  # fall through to brace scan
        # Try to find the first balanced JSON object/array
        # by scanning for { ... } or [ ... ] using a simple bracket counter
        for start_char, end_char in [("{", "}"), ("[", "]")]:
            start = text.find(start_char)
            if start == -1:
                continue
            depth = 0
            in_string = False
            escape = False
            for i in range(start, len(text)):
                c = text[i]
                if escape:
                    escape = False
                    continue
                if c == "\\":
                    escape = True
                    continue
                if c == '"' and not escape:
                    in_string = not in_string
                    continue
                if in_string:
                    continue
                if c == start_char:
                    depth += 1
                elif c == end_char:
                    depth -= 1
                    if depth == 0:
                        candidate = text[start : i + 1]
                        try:
                            return json.loads(candidate)
                        except json.JSONDecodeError:
                            break
        # Failed to extract — error
        raise OutputValidationError(
            errors=[{"type": "json_extract", "msg": "no parseable JSON found in response text"}]
        )
```

```python
# base.py
class RawChunk(BaseModel):
    index: int
    text: str
    finish_reason: str | None = None
    usage: dict[str, int] = Field(default_factory=dict)


# types.py
class Chunk(BaseModel):
    index: int
    text: str
    finish_reason: str | None = None
    usage: dict[str, int] = Field(default_factory=dict)


# client.py — in stream_text's terminal closure
async def terminal(r: Request, c: Context) -> AsyncIterator[Chunk]:
    async for raw in self._adapter.stream(r):
        yield Chunk(
            index=raw.index,
            text=raw.text,
            finish_reason=raw.finish_reason,
            usage=raw.usage,
        )


# observability.py — new on_stream_chunk override
async def on_stream_chunk(self, chunk: Chunk, ctx: Context) -> Chunk:
    if chunk.usage:
        # Accumulate per-key into ctx.metadata
        agg = ctx.metadata.setdefault("gen_ai.usage", {})
        for k, v in chunk.usage.items():
            agg[k] = agg.get(k, 0) + v
    return chunk


# local.py — emit usage on the final chunk + finish_reason="stop"
async def stream(self, req: Request) -> AsyncIterator[RawChunk]:
    full = (await self.generate(req)).text
    words = full.split(" ")
    total_input = len(prompt.split())  # whatever the prompt is
    total_output = len(words)
    for i, word in enumerate(words):
        await asyncio.sleep(0)
        is_last = i == len(words) - 1
        yield RawChunk(
            index=i,
            text=word + " ",
            finish_reason="stop" if is_last else None,
            usage={"input_tokens": total_input, "output_tokens": total_output} if is_last else {},
        )
```
